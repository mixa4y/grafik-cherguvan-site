# Графік чергувань

Публічний вебзастосунок для налаштування та формування графіків чергувань.

## Відкрити сайт

https://mixa4y.github.io/grafik-cherguvan-site/

Дані користувача зберігаються локально у браузері та не передаються до репозиторію.

## Основний принцип

Програма розділяє два різні поняття:

- **Людей у зміні** — загальний склад зміни.
- **Людей одночасно на пості** — скільки людей із цієї зміни потрібно конкретному посту.

Наприклад, у зміні може бути три людини, але на пост одночасно виходить лише одна. Тоді ці троє міняються між собою.

## Алгоритм роботи

### Крок 1. Загальні налаштування

Користувач задає:

1. Дату початку графіка.
2. Час початку службової доби.
3. Кількість днів.
4. Кількість постів.
5. Кількість змін.
6. Кількість людей у кожній зміні.

Програма автоматично створює:

- `Пост 1`, `Пост 2` тощо;
- `Зміна 1`, `Зміна 2` тощо;
- потрібну кількість порожніх рядків для людей.

### Крок 2. Налаштування постів

Для кожного поста окремо вказується:

1. Назва поста.
2. Тривалість одного чергування: `1, 2, 3, 4, 6, 8, 12 або 24 години`.
3. Години, коли пост не працює.
4. Які зміни можуть працювати на цьому пості.
5. Кількість людей одночасно на пості від кожної зміни.
6. Режим роботи змін:
   - **По черзі** — зміни міняють одна одну через задані проміжки.
   - **Одночасно** — усі вибрані зміни виходять разом.
7. Режим роботи людей усередині зміни:
   - **По черзі** — люди змінюють одна одну.
   - **Повним складом** — усі призначені люди виходять одночасно.

Одна зміна може бути призначена на кілька постів. На один пост також можна призначити кілька змін.

### Крок 3. Склад змін

Програма показує окремий блок для кожної зміни. Для кожної людини вводиться:

- підрозділ;
- ПІБ;
- номер телефону.

Стовпець вибору зміни не потрібен, оскільки людина вже знаходиться всередині блоку відповідної зміни.

Порожні рядки дозволяються. Після підтвердження програма попереджає, скільки людей не внесено, але дозволяє продовжити.

### Крок 4. Початкове призначення

Для першого дня визначається, хто куди виходить.

Доступні режими:

- **Автоматично** — програма розподіляє людей відповідно до потреб постів.
- **Вручну** — користувач сам призначає кожну людину.
- **Комбіновано** — програма робить розподіл, а користувач його коригує.

Якщо пост потребує всю зміну, вона призначається туди повним складом. Якщо потрібна лише частина зміни, люди розподіляються між постами або чергуються всередині поста.

### Крок 5. Правила ротації

Для кожної пари `зміна + пост` задається:

1. **Постійна** — зміна весь період залишається на цьому пості.
2. **Ротаційна** — зміна переходить на інші дозволені пости.
3. Частота переходу: кожні `1, 2, 3...` днів.
4. Рівень ротації:
   - переходить уся зміна;
   - переходять окремі люди;
   - люди змінюються тільки всередині поста.
5. Порядок переходу:
   - автоматично на наступний доступний пост;
   - за заданим маршрутом, наприклад `Пост 1 → Пост 3 → Пост 2`;
   - ручне призначення по днях.

Постійні зміни не беруть участі в загальній ротації.

### Крок 6. Автоматична перевірка

Перед формуванням графіка програма перевіряє:

- чи всі пости мають назви;
- чи кожному посту призначена хоча б одна зміна;
- чи вистачає людей у кожній зміні;
- чи не призначена одна людина одночасно на різні пости;
- чи не припадає чергування на неробочі години;
- чи вистачає людей для одночасної роботи;
- чи не конфліктують постійні та ротаційні призначення.

Критичні помилки блокують формування. Нестача людей показується червоним неоном, але користувач може продовжити.

### Крок 7. Формування графіка

Програма:

1. Розбиває кожну добу на проміжки.
2. Видаляє неробочі години постів.
3. Визначає активну зміну для кожного проміжку.
4. Застосовує постійні призначення.
5. Виконує ротацію через задану кількість днів.
6. Вибирає людей усередині зміни.
7. Рівномірно розподіляє навантаження.
8. Позначає нестачу та конфлікти.
9. Формує таблицю за днями, постами й часом.

За нестачі людей пропонуються варіанти:

- додати тимчасові значення `Черговий 1`, `Черговий 2`;
- перебудувати графік наявними людьми;
- повернутися до налаштувань.

## Результат

Сформований графік можна:

- переглядати на екрані;
- редагувати й перебудовувати без повторного введення даних;
- друкувати;
- експортувати в Excel.

Наприклад, зміна з трьох людей може повним складом постійно працювати на кухні, а інша зміна з трьох людей — одночасно закривати три пости по одній людині з внутрішнім чергуванням.

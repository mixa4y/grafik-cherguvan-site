# Майстер графіка чергувань

Публічний вебдодаток для поетапного створення постів, змін, складу наряду та графіка чергувань.

**Сайт:** https://mixa4y.github.io/grafik-cherguvan-site/

## Алгоритм роботи

### Крок 1. Початок

Відповідальна особа задає:

- перший день графіка;
- час початку службової доби;
- кількість днів графіка.

Люди та кількість змін на цьому кроці не задаються.

### Крок 2. Пости

Користувач обирає одну з дій:

- **Створити новий пост**;
- **Обрати існуючий пост** з локальної бібліотеки.

Новий пост відкривається повністю заповненим дефолтними значеннями: назва `Пост N`, дата й час із Кроку 1, одна або вже створені зміни, одна людина на зміну, 4 години та ротаційний режим. Усі значення можна відредагувати. Дата визначає, з якого дня пост з’являється у графіку, а початок доби — від якого часу будуються його проміжки чергування. Для кожного поста окремо задаються:

- назва поста;
- дата початку роботи поста;
- початок розрахункової доби поста;
- загальна кількість людей для поста;
- тривалість чергування: 1, 2, 3, 4, 6, 8, 12 або 24 години;
- режим роботи змін: по черзі або одночасно;
- зміни, які обслуговують пост;
- для кожної зміни: постійна робота або ротація між постами;
- проміжки простою, коли пост не працює.

Загальна кількість людей поста розподіляється між вибраними змінами і не дублюється в кожну зміну. Наприклад, якщо три пости потребують по дві людини та кожний пост обслуговують дві зміни, програма розрахує по одній людині від кожної зміни на пост: усього три людини у Зміні 1 та три людини у Зміні 2.

Зміну можна повторно використати на декількох постах або створити нову. Після збереження доступні дії **Дублювати пост**, **Створити новий** і **Далі**.

### Крок 3. Люди

Усі зміни автоматично підтягуються з постів Кроку 2. Потреба кожної зміни розраховується окремо за максимальним одночасним навантаженням на всіх постах.

Для кожної людини задаються:

- підрозділ;
- прізвище та ініціали;
- контактний номер телефону.

Рядки людей також мають видимі дефолтні значення: **Черговий N**, **Підрозділ не вказано**, **Не вказано**. Вони не вважаються підтвердженими персональними даними, тому програма чесно показує нестачу, доки користувач не замінить їх справжніми даними. Кожну зміну можна підтвердити окремо або підтвердити всі одразу. Незаповнені чи не замінені обов'язкові поля підсвічуються червоним; після попередження графік можна сформувати з позначеннями **Черговий N · Зміна N**.

### Крок 4. Початкове призначення

Перевіряється покриття постів у перший день. Доступний автоматичний, комбінований або ручний розподіл людей.

### Крок 5. Ротація

Для кожної зміни застосовуються правила, визначені в постах: постійна робота або перехід між доступними постами із заданою частотою.

### Крок 6. Графік

Перед побудовою показуються підсумок, нестача людей і можливі конфлікти. Після підтвердження формується таблиця чергувань.

## Додаткові можливості

- перемикач автозбереження: постійне відновлення даних або зберігання лише до закриття браузера;
- червоне підсвічування нестачі та конфліктів;
- кольорове розрізнення змін;
- друк готового графіка;
- експорт у Excel `.xlsx`;
- повне очищення даних;
- адаптивний інтерфейс DraculaMint.

## Запуск локально

Застосунок не потребує збірки. Відкрийте `index.html` у браузері або запустіть локальний сервер у цій папці.

У PowerShell:

```powershell
.\serve.ps1 -Port 8767
```

Після цього відкрийте `http://localhost:8767/`.
